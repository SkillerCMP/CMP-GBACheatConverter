# EZ-Flash Omega DE Enhanced v3 compatibility

The converter exposes two EZ-Flash modes:

- **Original** — stock-compatible `ON=` byte writes only.
- **Enhanced** — Omega DE Kernel 1.06 Enhanced v3.

Enhanced keeps comma-separated byte lists. There are no W8/W16/W32 commands.
Bytes are little-endian when interpreted numerically, and write/condition
payloads may exceed 32 bits.

## Runtime writes and conditions

```ini
[Eight Byte Write]
ON=10A78,01,02,03,04,05,06,07,08;

[Compare With Else]
IFNE=202,01,00;ON=10A78,CC;ELSE;ON:10A78,64;ENDIF;
```

Supported condition keys are `IF=`, `IFNE=`, `IFLT=`, `IFGT=`, `IFLE=`, and
`IFGE=`. Ordered comparisons are unsigned byte-array comparisons. `ELSE` and
`ENDIF` are bare semicolon-delimited tokens. Existing sequential groups and
lines without explicit `ENDIF` remain readable.

## Arithmetic

```ini
[Wide Arithmetic]
ADD=10A78,01,02,03,04,05,06;SUB:10A80,01,00;
```

ADD/SUB carry or borrow across all listed bytes and execute each engine tick.
Top-level `ADD=`/`SUB=` and inline `ADD:`/`SUB:` are accepted.

## Pointer write

```ini
[Pointer Write]
PTR=10,00000123,7F;
```

Grammar: `PTR:<compact-pointer-address>,<32-bit offset>,<payload bytes>`. The
pointer source and final EWRAM/IWRAM target must pass the kernel range checks.

## Fill and slide

```ini
[Fill]
FILL=10000,00000003,34,12;

[Slide]
SLIDE=10020,00000004,00000002,FFFFFFFF,34,12;
```

FILL grammar is `address,repeat-count,pattern-bytes`. SLIDE grammar is
`address,count,signed-address-step,signed-value-step,initial-bytes`. Both are
expanded transactionally into byte writes and therefore consume runtime
records for every generated byte.

## ROM support

```ini
[ROM Patch]
ROM=080EA0C0,00,20,70,47;

[Guarded Mixed Patch]
ROMIF=09014CD3,FD,00;ON=239F8,04;ROM=080EA0C0,00,20;

[Runtime IF Plus Independent ROM]
IF=80130,FF,00;ON=239F8,04;ROM=0803A830,00,20;
```

`ROMIF=` is a pre-launch byte-array equality guard. A trailing ROM patch after
a runtime IF is independent of that runtime condition. Legacy inline `ROM:`
and `ROMIF:` forms remain readable; canonical output prefers top-level forms.

## Compact address map

- `00000-3FFFF` -> EWRAM `02000000-0203FFFF`
- `40000-47FFF` -> IWRAM `03000000-03007FFF`
- `80000-803FF` -> I/O `04000000-040003FF` for condition reads only

ROM runs accept full Game Pak addresses and are emitted canonically as full
addresses.

## Runtime costs and limits

The Enhanced kernel keeps one shared 128-record runtime table. Direct bytes,
condition headers/bytes, ELSE/ENDIF markers, arithmetic bytes, pointer control
records, and generated FILL/SLIDE bytes consume runtime records. ROMIF and ROM
bytes use separate pre-launch tables.

The converter also enforces 49-byte section names and 298 visible characters
per physical line. Malformed or overflowing entries are suppressed as a
complete unit so dependent writes cannot leak.

## Converter mappings

- CodeBreaker/Xploder slides map to `SLIDE=` when exact.
- GameShark/AR GBX NE/unsigned LE/GE and arithmetic forms map to Enhanced
  comparisons or ADD/SUB when exact.
- AR MAX signed/unsigned comparisons, IF/ELSE blocks, one-level pointers,
  fill/slide operations, and audited direct ROM patches map when exact.
- Device-only slowdown, hook/master, interception, or unsupported nested
  semantics remain suppressed with a compatibility note.
